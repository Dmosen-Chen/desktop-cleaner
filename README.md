# 桌面工作区（简洁版）

在 Windows 桌面上提供半透明分组工作区，用来收纳文件、文件夹和快捷方式的**入口引用**；从面板菜单添加入口不会移动源文件。界面由 Python **tkinter** 提供。不包含壁纸、AI、云同步或网络功能。

## 功能概览

- 自动探测常见桌面路径（含 `Desktop`、`桌面`、OneDrive 桌面等）
- 自定义规则：**后缀 → 归档子文件夹名**
- **预览整理**：正式移动前先查看将移动的文件与目标位置
- **一键整理**：只处理桌面上匹配规则的**文件**（默认不移动文件夹）
- **撤销上一次整理**：按批次把文件移回桌面；若遇到冲突，会保留未完成撤销记录以便重试
- **系统托盘**（需安装依赖）：关闭窗口可最小化到托盘；托盘菜单支持一键整理等
- **开机启动**：写入当前用户「启动」文件夹快捷方式
- **桌面工作区**：主显示器工作区内半透明置顶区域；从面板 `...` 菜单添加文件、文件夹或路径 → 只添加快捷入口，不移动源文件。直接拖入为 `windnd` 实验功能，默认关闭
- **入口操作**：双击入口打开原路径；右键可从工作区移除入口，源文件不会被删除
- **工作区编辑**：标题栏保留锁按钮、名称和 `...` 菜单；双击名称可直接重命名，解锁后可移动和拖右下角调整大小
- **外观设置**：可在 `...` 菜单中选择面板颜色和透明度
- **面板小组**：拖动面板时鼠标落入另一个面板范围即可合并；单击标签切换内容，拖动标签可把该面板拆出小组
- **按规则收纳入口**：在 `... → 整理规则` 中设置扩展名，点击「一键整理桌面」会把桌面上匹配的文件添加到对应面板，不移动源文件
- 配置持久化到用户目录，与仓库代码分离

## 环境要求

- Windows
- **Python 3.10+**（安装时勾选或另行安装 **tcl/tk**，以便使用 `tkinter`）

### Python 依赖

完整功能（托盘图标 + 托盘菜单）需要：

```bash
pip install -r requirements.txt
```

（安装 **pystray**、**Pillow**、**windnd**。若不装托盘依赖仍可整理；`windnd` 默认不启用，需要测试直接拖入时设置 `DESKTOP_TIDY_ENABLE_WINDND=1`。）

**产品与范围说明**（功能列表、非目标、里程碑）见 [`docs/需求说明.md`](docs/需求说明.md)。

## 快速开始

```bash
pip install -r requirements.txt
python main.py
```

首次保存配置或整理前若未指定归档路径，默认归档根目录为：

`%USERPROFILE%\Documents\桌面整理`

可在界面中点击「浏览…」修改为任意文件夹。

## 配置文件位置

| 文件 | 路径 |
|------|------|
| 用户配置 | `%LOCALAPPDATA%\DesktopTidy\config.json` |
| 撤销记录 | `%LOCALAPPDATA%\DesktopTidy\undo_stack.jsonl` |

仓库内 `config.default.json` 提供初始规则模板；首次运行会与用户配置合并（用户已有规则优先生效）。

在 `config.json` 的 `ui.partitions` 中可调整：`name`（显示）、`items`（入口列表，只保存路径引用）、颜色及相对位置（`rx`/`ry`/`rw`/`rh` 为相对主工作区的 0–1 比例）。

也可以直接在界面中点击「管理工作区」进行新建、重命名或删除；双击工作区名称可在标题栏内直接编辑。点击工作区标题栏的锁图标可切换锁定/移动状态，解锁后显示 `🔓`，拖动标题栏或面板空白处即可移动，拖右下角 `◢` 可调整大小，松开鼠标会自动保存布局。其它操作集中在标题栏的 `...` 菜单里，包括新建面板、整理规则、颜色和透明度。新建面板会直接出现在桌面上，并自动进入名称编辑。

## 使用注意

- 当前「一键整理桌面」只会按设置中心规则把桌面项目添加到面板入口，不会移动、复制或删除源文件。
- 建议先使用 **预览整理** 确认入口列表；点击「一键整理桌面」时也会再次确认。
- 从其它路径添加文件时，会在配置的桌面路径里创建 `.url` 入口，源文件位置不变。
- **撤销**仅针对「最后一次一键整理」生成的移动批次；多次操作请依次撤销（先进后出）。若原位置已有同名文件，程序会跳过该文件并保留撤销记录，处理冲突后可再次撤销。工作区入口移除不会删除源文件。
- **桌面工作区**：添加任意文件、文件夹或快捷方式只会添加入口，不会移动、复制或删除源文件。「一键整理桌面」也只会按面板整理规则添加入口，不会移动源文件。
- 若桌面快捷方式指向已移动的目标文件，可能出现路径失效，需自行更新快捷方式。

## 打包为 exe（正式交付步骤）

在未安装 Python 的电脑上使用时，在本机（需已安装 Python）生成 **单文件、无黑框控制台** 的可执行文件。

1. 确认：`pip install -r requirements-build.txt` 后 **`python main.py`** 运行正常。
2. 执行：

   ```bash
   scripts\build_exe.bat
   ```

   或在项目根目录使用与脚本等价的 PyInstaller 命令（含 `--collect-all=pystray`、`--collect-all=PIL` 等，详见 `scripts\build_exe.bat`）。

3. 产物：**`dist\DesktopTidy.exe`**。用户配置仍写入 `%LOCALAPPDATA%\DesktopTidy\`。

说明：`--add-data` 将默认规则模板打进 exe；程序通过 `_MEIPASS` 读取内置资源（见 `main.py` 中 `_bundle_root()`）。

## 仓库结构（简要）

```
tool/
├── main.py                 # GUI 入口（托盘 / 分区）
├── organizer.py            # 整理、撤销、路径与配置逻辑
├── partition_overlay.py    # 桌面分区悬浮窗
├── tray_support.py         # 系统托盘（可选依赖）
├── windows_native.py       # 工作区、穿透点击、启动快捷方式
├── config.default.json     # 默认规则模板（打包时嵌入 exe）
├── requirements.txt        # 运行时推荐依赖（pystray、pillow）
├── requirements-build.txt    # 打包：含上述依赖 + PyInstaller
├── scripts/
│   └── build_exe.bat       # Windows 一键打包
├── docs/
│   └── 需求说明.md
└── README.md
```

# Product

## Register

product

## Users

Windows users who want a calmer desktop without changing where their real files live. The main usage context is daily desktop work: launching files, folders, shortcuts, and lightweight widgets from transparent panels placed on one or more monitors.

## Product Purpose

DesktopCleaner is a Windows desktop entry manager. It displays desktop items, virtual external references, icon groups, and widget panels in configurable desktop panels. The product succeeds when users can organize and launch entries quickly while the app preserves the safety boundary: it does not move, copy, or delete source files.

## Brand Personality

Quiet, technical, desktop-native. The interface should feel like a focused Windows utility with a subtle translucent desktop character, not a loud launcher skin or a marketing surface.

## Anti-references

Avoid copying third-party desktop organizer branding, mascot assets, wallpapers, or proprietary skin elements. Avoid over-decorated glassmorphism that makes text hard to read. Avoid engineering-table interfaces when the user is making visual layout or appearance decisions.

## Design Principles

1. Preserve source files: all organization features should clearly operate on app metadata unless explicitly stated otherwise.
2. Make state visible: desktop takeover, grouping, updates, and recovery states must be understandable before the user acts.
3. Keep common desktop behavior familiar: opening, right-clicking, selecting, resizing, and tray behavior should follow Windows expectations where practical.
4. Use translucency with restraint: panel identity can be transparent, but text, controls, and decision surfaces must remain readable.
5. Separate concepts cleanly: desktop panels, icon groups, widget panels, classification rules, and diagnostics should not share ambiguous labels or data paths.

## Accessibility & Inclusion

Target readable contrast for normal text and controls. Avoid relying on color alone for selection or warnings. Support reduced-motion-safe transitions and keep animations short enough that repeated desktop operations do not feel delayed.

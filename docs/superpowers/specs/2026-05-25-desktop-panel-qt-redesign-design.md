## 1. 摘要

本项目重构为 Windows 桌面层的入口管理器：在桌面背景层上展示可分组、可整理、可定制的入口面板，让用户用轻量、直观的分组面板管理桌面内容。

Qt v1 借鉴常见桌面整理工具的通用交互模式，但不复制任何第三方代码、皮肤资源、品牌元素或壁纸能力。新界面使用 `PySide6` 全面替换当前 Tkinter 面板窗口实现；可复用既有项目中合理的配置与分类思想，但不延续当前不稳定的窗口、拖放和真实文件整理路径。

Qt v1 的核心原则是：显示和分类入口，不移动、复制或删除源文件。真实文件归档将来若开发，必须作为独立的高级模块出现，不属于本版本范围。

## 2. 产品边界

### 2.1 Qt v1 目标

- 在主显示器桌面层展示面板，而不是浮在普通应用窗口之上。
- 接管启用时，以面板内的分类入口替代 Explorer 原桌面图标的视觉展示。
- 自动索引用户配置的桌面目录，并按照规则即时分类展示桌面项目。
- 允许用户创建、删除、重命名、合并、拆分和调整面板标签组。
- 允许将桌面外文件或文件夹加入面板，作为仅存在于本应用配置中的虚拟引用。
- 提供稳定、可恢复的 Windows 桌面接管和最终无控制台单文件 `exe` 交付。

### 2.2 Qt v1 明确不包含

- 不移动、复制、重命名或删除桌面及外部路径中的源文件。
- 不提供“一键整理到文件夹”、真实归档、撤销文件移动或归档目录管理。
- 不为外部拖入项目创建桌面快捷方式、`.url` 文件或任何真实桌面副本。
- 不提供搜索、壁纸、AI、云同步、账号同步或半完成状态的入口。
- 不渲染多显示器面板；数据结构仅预留未来扩展字段。

### 2.3 后续预留

未来可单独设计“真实文件归档”高级模块。该模块必须使用明确区分于入口分类的命名、操作确认、撤销和风险提示，不能复用 Qt v1 中的普通分类按钮制造“显示分类等于移动文件”的误解。

## 3. 用户体验与交互

### 3.1 桌面接管

- 用户启用桌面接管后，应用首先尝试成功挂载到 Windows 桌面层，再隐藏 Explorer 原桌面图标。
- 面板仅位于桌面层：正常应用窗口应自然显示在其上方。
- Qt v1 仅在主显示器渲染面板，所有新建组默认属于主显示器。
- 用户关闭桌面接管、退出程序、启动失败或程序执行恢复流程时，必须恢复 Explorer 原桌面图标。
- 若无法创建桌面层面板，应用不得隐藏原桌面图标，应提示接管失败并提供非接管状态下的设置入口。

### 3.2 首次启动默认状态

- 首次启动或旧配置迁移完成后，自动建立一个 `PanelGroup`。
- 默认组内依次包含六个标签：`文件夹`、`文档`、`图片`、`压缩包`、`应用`、`其它`。
- 应用直接读取已配置桌面路径中的当前项目，按默认分类规则展示；不要求用户先预览或确认。
- 默认面板外观为黑色背景，背景 `alpha = 0.60`。本文将用户所说的“透明度 60”固定解释为背景不透明度 `60%`，避免实现阶段得到相反的视觉效果；背景透明效果不影响图标和文字的不透明渲染。
- 默认布局状态为解锁，允许用户立即移动、缩放和重组面板。

### 3.3 面板组与标题栏

每个可见组使用统一标题栏和统一外观。标题栏包含：

- 左侧：收起/展开按钮、锁定/解锁按钮、当前标签名称。
- 右侧：`+` 新建标签按钮、垃圾桶删除当前标签按钮、`...` 其它操作菜单。

行为规则如下：

- 点击当前标签名称可直接进入原位重命名，不弹出独立输入对话框。
- 点击 `+` 在当前组内新建空白标签，并立即进入标签名称编辑状态。
- 点击垃圾桶删除当前标签。若该标签是组内最后一个标签，必须先确认删除整个组。
- 系统始终保留至少一个默认面板组；删除最后剩余组时应重建或保留一个默认组。
- 删除标签或面板组仅删除应用内布局与引用关系，不得删除、移动或复制任何源文件；受影响的真实桌面项目重新进入自动分类。
- 搜索、壁纸、AI、同步、真实归档等未提供的功能不得以禁用按钮形式出现在标题栏或菜单中。

### 3.4 标签切换、拆分与合并

- 点击组内标签时，立即切换当前显示内容；标题、项目列表和选中状态随之更新。
- 按下标签并拖动时，原大面板保持原位，鼠标附近显示被拖标签的预览反馈。
- 松开鼠标于原组外时，该标签从原组删除并形成一个独立面板组；若原组已无标签，按删除组规则处理。
- 拖动独立面板到另一面板组的可合并区域内，依据松开时鼠标落点合并为同一组，而不是依赖难以理解的窗口边界碰撞。
- 合并后，各标签继续保存各自名称、项目覆盖关系和排序；整个新组使用同一份组级外观设置。

### 3.5 移动、锁定、收起和缩放

- 解锁状态下，拖动标题栏移动整个面板组。
- 锁定状态下，不允许面板移动、边缘缩放、标签拆出或被拖入合并；标签切换和打开项目仍可使用。
- 收起按钮只折叠项目内容和标签区域，保留标题栏，并保持按钮在相同层级和稳定位置；收起后不得残留多余底边或空内容条。
- 解锁状态下，可从水平边缘调整宽度、垂直边缘调整高度，并允许使用角落同时调整宽高。
- 移动、收起、锁定、缩放、标签排序和合并结果应及时持久化，重新启动后恢复。

### 3.6 外观与项目渲染

- 每个 `PanelGroup` 只有一套颜色和透明度；同组下所有标签显示相同外观。
- 面板背景可透明，图标、图片缩略图和文字必须保持不透明且清晰可读。
- 普通文件、文件夹和应用入口使用 Windows 原生图标；图片项目显示缩略图。
- 项目以自动网格布局展示，窗口尺寸变化后自动重新排布。
- 文件名最长显示两行，超出区域时使用省略显示，不能挤压相邻图标或超出面板。
- 空标签显示简洁空状态，提供添加虚拟引用和进入分类规则设置的入口。

### 3.7 项目行为

- 桌面目录中的真实项目只作为可打开的入口显示；打开项目不改变其磁盘位置。
- 任意格式的文件以及文件夹均可被拖入面板；格式不匹配现有规则的桌面项目归入 `其它`。
- 从桌面外拖入文件或文件夹时，应用仅保存虚拟引用，不创建桌面快捷方式、不复制也不移动源项目。
- 虚拟引用的源路径不存在时，显示失效状态，并允许用户重新关联或从面板移除引用。
- 将项目拖到另一标签会创建手动分类覆盖；普通同步和重新执行自动规则不得移走该项目。
- 用户可对某个项目选择“恢复自动分类”，删除其手动覆盖并再次按规则分配。

## 4. 桌面索引、分类与同步

### 4.1 数据来源

- 用户设置中的桌面路径是桌面真实项目的唯一目录来源，支持用户输入自定义路径，例如重定向后的 Windows 桌面目录。
- 配置文件不保存桌面目录中所有真实项目的静态副本；运行时由目录索引产生当前列表。
- 配置文件只保存布局、规则、手动覆盖、外部虚拟引用、外观和桌面接管状态。

### 4.2 默认分类规则

默认规则按以下目标标签分类，并允许用户在设置中心启用、禁用、编辑后缀或改变目标标签：

| 类型 | 默认匹配方式 | 默认目标标签 |
| --- | --- | --- |
| 文件夹 | 目录 | 文件夹 |
| 文档 | 常用文本、Office、PDF 及文档后缀 | 文档 |
| 图片 | 常用图片和设计图像后缀 | 图片 |
| 压缩包 | 常用压缩与归档后缀 | 压缩包 |
| 应用 | 快捷方式、可执行入口及应用链接 | 应用 |
| 其它 | 不属于以上已启用规则的项目 | 其它 |

默认情况下，系统元数据及常见临时文件，例如 `desktop.ini`、隐藏/系统项目以及 Office 临时文件，应从普通面板展示中排除，避免污染首次使用体验。

### 4.3 分类优先级

一个项目的目标标签按以下顺序确定：

1. 已存在且有效的 `ManualOverride`。
2. 外部虚拟引用中明确保存的目标标签。
3. 第一条启用并匹配的 `ClassificationRule`。
4. 默认 `其它` 标签。

规则改变后，仅重新分类没有手动覆盖的桌面真实项目；外部虚拟引用除非用户主动改变目标标签，否则保持所在位置。

### 4.4 实时同步

- 应用监听已配置桌面目录的新增、删除和重命名事件，并实时刷新当前面板。
- 新增真实桌面项目立即按分类优先级显示。
- 删除或改名不会触发对磁盘文件的额外写入；失效的手动覆盖可在刷新时清理或提示清理。
- 目录监听暂时失败时，面板保留最后一次成功索引结果并提示刷新失败，绝不以文件移动作为修复手段。

## 5. 设置中心

Qt v1 设置中心只展示已经可用且属于入口管理器范围的设置：

- 基础设置：开机启动、退出行为和必要的恢复状态提示。
- 桌面分区：桌面路径输入与选择、桌面接管开关。
- 桌面整理：分类规则列表、规则开关、后缀配置、目标标签选择、恢复默认规则。
- 面板外观：默认组颜色、透明度以及新建组使用的默认外观。

设置中心不显示归档目录、真实移动、撤销归档、搜索、壁纸、AI、云同步或其它未在 Qt v1 实现的功能入口。

## 6. 技术方向与核心数据接口

### 6.1 UI 和平台架构

- 交付界面使用 `PySide6` 全面替换 Tkinter UI。
- 当前 Tkinter 面板窗口、透明窗口叠层和不稳定拖放实现不作为新 UI 的实现基础。
- Windows 集成封装为独立服务层，负责桌面层挂载、Explorer 图标可见性恢复、系统图标/缩略图获取、开机启动和桌面目录监听。
- UI 层依赖领域模型和 Windows 服务接口，不直接散布 Win32 恢复逻辑。

### 6.2 配置位置与版本

- 新配置继续保存于 `%LOCALAPPDATA%\DesktopTidy\`。
- 新配置应包含明确的 `schema_version`，Qt v1 使用新的版本号以触发迁移。
- 面板位置应保存可适配屏幕工作区域的坐标与尺寸，并预留 `screen_id`；Qt v1 仅读取和渲染主显示器对应布局。

### 6.3 核心实体

#### `PanelGroup`

代表一个可以移动、缩放、收起和锁定的大面板组。

- `id`
- `screen_id`
- `geometry`：位置与尺寸
- `tab_ids`：有序标签列表
- `active_tab_id`
- `appearance_id` 或内嵌组级外观
- `locked`
- `collapsed`

#### `PanelTab`

代表组内的一个可切换小面板。

- `id`
- `group_id`
- `name`
- `order`
- `category_role`：默认分类标签或自定义标签

#### `ItemRef`

代表面板可展示和打开的项目引用。

- `id`
- `source_kind`：`desktop` 或 `external`
- `canonical_path`
- `target_tab_id`：仅外部虚拟引用必须持久保存
- `display_state`：运行时计算的有效或失效状态

其中 `desktop` 类型由实时目录索引产生，不作为项目列表持久化；`external` 类型作为虚拟引用保存到配置。

#### `ClassificationRule`

代表自动分类规则。

- `id`
- `name`
- `enabled`
- `matcher_kind`
- `extensions` 或其它匹配配置
- `target_tab_id`
- `order`

#### `ManualOverride`

代表用户对桌面真实项目作出的手动归类决定。

- `canonical_path`
- `target_tab_id`

它优先于自动分类规则，直到用户明确选择恢复自动分类。

#### `AppearanceSettings`

代表组级统一样式及新建组默认样式。

- `background_color`
- `background_opacity`：背景 alpha 值

Qt v1 默认值为黑色背景及 `background_opacity = 0.60`。

#### `DesktopIntegrationState`

代表桌面接管与异常恢复相关的状态。

- `desktop_path`
- `takeover_enabled`
- `explorer_icons_hidden`
- `restore_required`
- `startup_enabled`
- `primary_screen_id`

## 7. 旧配置迁移

### 7.1 迁移目标

旧配置中已有面板映射、规则分配和项目列表已经产生混乱展示，不能直接沿用为 Qt v1 布局来源。迁移应以获得干净、可理解的首次体验为目标。

### 7.2 迁移步骤

1. 在写入新格式前，复制旧 JSON 为同目录带时间戳的备份文件，例如 `config.pre-qt-v1-YYYYMMDD-HHMMSS.json`。
2. 保留有效的桌面路径和开机启动偏好。
3. 若旧配置包含可验证的统一颜色和背景 alpha，则可将其导入为新默认组外观；值无效或不一致时使用黑色、`background_opacity = 0.60`。
4. 对可明确识别为桌面外路径的旧项目引用，导入为 `external` 虚拟引用，并默认放入 `其它` 标签。
5. 创建新的单一默认组和六个默认标签，按实时桌面索引重新生成当前展示。

### 7.3 不迁移内容

- 不迁移旧面板 ID、旧分组关系、混乱的标签映射和旧分类分配。
- 不迁移归档目录、真实文件移动记录或撤销历史。
- 不将旧 `.url` 或快捷方式作为新迁移副作用写入真实桌面。

迁移失败时保留备份与原配置，应用应向用户提示恢复或重试，而不是以空配置覆盖可恢复信息。

## 8. Windows 集成与故障安全

### 8.1 接管启动顺序

1. 读取恢复标记；若上次运行遗留 `restore_required` 或图标隐藏状态，启动阶段优先尝试恢复 Explorer 图标。
2. 初始化配置、面板模型和目录索引。
3. 创建并验证桌面层面板挂载成功。
4. 仅在第 3 步成功后，先持久化恢复标记，再隐藏 Explorer 原桌面图标，以覆盖隐藏操作期间异常退出的恢复风险。
5. 正常关闭或禁用接管时，先恢复 Explorer 图标，再清除恢复标记。

### 8.2 故障处理

- 桌面层挂载失败：不隐藏图标，提示失败并允许用户进入设置中心。
- 接管过程异常：尽最大努力立即恢复图标，并保留恢复标记供下次启动再次恢复。
- 配置损坏：保留损坏文件副本，初始化干净默认布局并提示用户。
- 项目路径失效：只标记入口失效，不删除或移动任何文件。

恢复桌面显示能力属于发布阻断级要求：任何可能让用户退出后仍看不到 Explorer 桌面图标的路径，都不得作为可交付版本发布。

## 9. 交付与验证

### 9.1 发布交付

- Qt v1 最终生成单文件、无控制台窗口的 Windows `exe`。
- 打包应包含所需 Qt 平台插件、应用图标、默认规则资源和版本化迁移能力。
- 发布包不得包含从参考软件提取的代码、皮肤或品牌资源。

### 9.2 自动测试覆盖

- 旧配置备份与迁移。
- 默认规则分类和自定义规则优先顺序。
- 手动覆盖创建、保持和恢复自动分类。
- 外部虚拟引用持久化与失效状态。
- 配置读写、默认外观和布局持久化。
- 桌面目录新增、删除、重命名事件映射为视图模型更新。
- 桌面接管状态机的恢复标记流程，以可替代的 Windows 服务接口进行测试。

### 9.3 人工冒烟验收清单

- 首次启动创建一个默认标签组，并正确展示、分类当前桌面项目。
- 自定义桌面路径可用于重定向桌面目录，刷新与实时同步正常。
- 从桌面外加入项目时只产生虚拟引用，磁盘和真实桌面内容不改变。
- 手动把项目移至另一标签后，重启和普通规则刷新均保留该位置；恢复自动分类生效。
- 标签点击切换、原位重命名、新建、删除、拆出、组间合并均符合交互设计。
- 面板锁定、收起、标题栏移动、水平/垂直边缘缩放和重启恢复均正常。
- 同组标签拥有同一背景颜色和透明度，图标与文字保持不透明，原生图标、图片缩略图和长名称省略显示正确。
- 桌面接管启用时图标隐藏，禁用或正常退出后图标恢复。
- 模拟挂载失败和异常退出后，下次启动能恢复 Explorer 桌面图标。
- 单文件无控制台 `exe` 在干净 Windows 环境可以启动、配置、退出并完成恢复。

## 10. 当前阶段与后续流程

本文件是 Qt 重构的需求与设计边界文档。当前阶段只建立文档，不修改现有 Tkinter 程序逻辑，不初始化 Git 仓库，也不生成新的 `exe`。

后续流程为：

1. 用户审阅并确认或修订本设计文档。
2. 文档确认后，编写具体的 PySide6 重构实施计划，包括模块切分、迁移顺序、测试策略和打包步骤。
3. 实施计划再次获得确认后，才开始 Qt 功能重写。

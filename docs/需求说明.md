# 桌面工作区（简洁版）— 产品需求说明



本文档用于在项目早期固定 **做什么 / 不做什么**，后续迭代以此为准；实现细节可在 Issue 或 PR 中补充。



---



## 1. 产品定位



一款 **仅在本地运行** 的 Windows 小工具，在桌面上提供半透明工作区面板，用于收纳文件、文件夹、快捷方式和程序的 **入口引用**；拖入工作区时只保存路径，不移动源文件。按后缀归档整理仍作为辅助功能保留。  

**不包含** 壁纸、AI、账号登录、云同步、广告或联网依赖。



---



## 2. 目标用户与典型场景



| 场景 | 说明 |

|------|------|

| 桌面入口太多 | 希望把常用文件、文件夹、软件快捷方式收纳到几个分组面板 |

| 不想改变文件位置 | 只想管理入口引用，不希望工具移动或删除源文件 |

| 小白用户 | 规则可视化编辑，无需手工写脚本 |

| 常驻后台 | 关闭主窗口后仍可从托盘一键整理；开机自动启动 |

## 3. 功能需求



### 3.1 必须实现（v1.0 核心）



| ID | 需求 | 验收要点 |

|----|------|----------|

| F1 | **桌面路径检测** | 在常见路径（如 `Desktop`、`桌面`、OneDrive 桌面）中自动选用存在的目录；界面展示当前使用的桌面路径 |

| F2 | **归档根目录** | 用户可指定任意本地文件夹为归档根目录；未指定时使用默认路径（见 README） |

| F3 | **规则：后缀 → 子文件夹** | 支持多条规则；后缀统一为小写并带点（如 `.pdf`）；同一后缀对应一个目标子文件夹名 |

| F4 | **规则持久化** | 配置写入用户目录 JSON，与程序安装位置无关；自带默认规则模板供首次合并 |

| F5 | **一键整理** | 扫描桌面上的 **文件**（默认不处理子文件夹）；命中规则则移动到 `归档根目录/子文件夹/` |

| F6 | **同名冲突** | 目标已存在同名文件时 **不得覆盖**，应自动改名（如追加随机后缀）并记录日志 |

| F7 | **撤销上一次整理** | 针对最近一次 **一键整理** 生成的移动批次，按相反顺序移回原路径；工作区入口移除不影响源文件 |

| F8 | **操作日志** | 界面内展示整理与撤销的关键日志文本，便于用户自查 |



### 3.2 应该具备（exe 交付）



| ID | 需求 | 验收要点 |

|----|------|----------|

| F9 | **打包为 Windows exe** | 按 README 构建后得到 **单文件、无控制台窗口** 的可执行文件；内置默认规则；可选依赖（托盘）打进包内 |

| F10 | **文档** | README 中含运行方式、依赖、配置路径、打包步骤与风险提示 |



### 3.3 v1.1 增强（已实现）



| ID | 需求 | 验收要点 |

|----|------|----------|

| B3 | **系统托盘** | 安装 `pystray`、`Pillow` 后：托盘图标可用；菜单含打开主窗口、一键整理、退出；可与「关闭时最小化到托盘」配合 |

| B3b | **开机启动** | 勾选后在当前用户 **启动文件夹** 创建/删除 `DesktopTidy.lnk`；exe 与源码模式均可指向正确目标 |

## 4. 配置说明（`ui` 与工作区）



配置文件中 `ui` 字段（节选）：



- `minimize_to_tray`：关闭主窗口时是否最小化到托盘（无托盘依赖时无效）。

- `overlay_visible`：是否显示桌面工作区。

- `startup_enabled`：与启动文件夹快捷方式同步（以实际 `.lnk` 是否存在为准）。

- `partitions`：工作区数组。每项包含：`name`、`items`（入口引用列表，只保存路径）、`rx`、`ry`、`rw`、`rh`（相对主工作区的 0–1 比例）、可选 `color`、`alpha`。



---



## 5. 非功能需求



| 类型 | 要求 |

|------|------|

| 隐私与安全 | 不向任何远程服务器上传文件内容或目录列表；配置仅存本地 |

| 性能 | 单次整理在常见桌面文件数量（数百级）下应在秒级完成 |

| 兼容性 | 目标：**Windows 10/11**，64 位；Python 3.10+（源码运行） |

| 可维护性 | 文件移动、撤销、路径解析与 GUI **分离**（核心逻辑可单元测试） |

## 6. 明确不做的内容（非目标）



- 壁纸、主题、动态桌面  

- AI 对话、智能推荐、文档解读  

- 账号体系、云同步、多端协作  

- 自动下载更新（若将来需要，单独评估）  

- 内置文件预览、重复文件清理（可作为独立工具，非本产品必选）



---



## 7. 风险与用户提示（需在界面或 README 中可见）



- 整理会 **真实移动文件**，可能影响依赖「固定桌面路径」的旧快捷方式或脚本。  

- **撤销** 只覆盖「上一次整理」批次；更早操作需依赖用户自行备份。  

- **桌面工作区** 通过 **拖放** 添加入口引用，不会移动源文件；「一键整理」仍按 **后缀规则** 批量处理桌面，属于辅助功能。  

- 工作区窗口会挡住其下方桌面区域；若要点击被挡住的图标，可先取消勾选「显示桌面工作区」。
